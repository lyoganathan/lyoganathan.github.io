Linear Regression with Continous and Categorical Variables

Linear Regression:

There are different types of regression. The most common is linear regression. Wikipedia can explain way better: https://en.wikipedia.org/wiki/Linear_regression and https://en.wikipedia.org/wiki/Simple_linear_regression.

Consider this formula from highschool: y=mx+b. Given some data x and y, linear regression will find the slope(m) and y-intercept(b) of a line that minimizes the sum of squared residuals. Residuals in this case are the vertical distance between the data point and the line you are trying to find. Put another way, it will find a line that minimizes the squared distance between each point and the line. The most common method used to find parameters that minimize squared residuals is called ordinary least squares (OLS).

Linear Regression where x and y are continuous:

We will first look at regression where the independent variable (x) and dependent variable (y) are both continous. For this we will group by the call date and take the count of calls.

numCalls = d[, list(NumCalls=length(WordCount)), by = (Date=as.Date(ClientCaptureDate))]
# Order data by date:
numCalls = numCalls[order(numCalls$Date)]

Plot:

plot(numCalls)

Do some linear regression:

Now lets try and find the line of best fit. First we will pass the data to lm which can find the slope and intercept parameters for us:

simple_model = lm(NumCalls ~ Date, data = numCalls)

Normally youd want to change the origin or scale the date in some way but we will not explore that here.

Plot the line we just found:

So the lm found our line of best fit and it is stored in the simple_model variable. If we want to plot our model on top of our data, first we use the model to predict y for a range of xs, and then we plot that.

# Predict
newx = seq.Date(from=as.Date('2021-06-01'),to=as.Date('2021-07-01'),by="days")
predy = predict.lm(simple_model, newdata = list(Date = newx))

plot(x=numCalls$Date,y=numCalls$NumCalls)
lines(newx,predy,col='blue')

A simpler way to do the above is just to write this:

plot(x=numCalls$Date,y=numCalls$NumCalls)
lines(numCalls$Date, predict(simple_model),col='blue')

The model is a pretty terrible fit as our data seems to go up and down a lot.

Linear Regression where y is continuous and x is categorical with 2 categories:

A categorical variable can also be called a factor. The number of categories is called levels. So in this case we can also say x is factor with 2 levels.

Look at just 2 levels:

We will start by just selecting 2 of the call queues:

test_df = d[d$UDF_text_07 == c('Sales','Retention'), c("UDF_text_07","WordCount")]
# Removing unused factor level:
test_df$UDF_text_07 = factor(test_df$UDF_text_07)

Lets look at some summary metrics

In this case the intercept and slope are much more readable and have meaning. If you notice, the intercept is actually the mean of the Retention group. And if you add the intercept and the slope you get the mean of the Sales group. What kind of sorcery is this?

Lets think about how a categorical variable fits into y=mx+b. Your x in this case can be two values: Retention and Sales. We cant really solve our equation if it has words in it. So, what R does internally is one hot encoding. Your x can either be 1 or 0, and that maps back to your categories. Your y-intercept occurs when x is 0, so the y-intercept is the mean of the group which got x = 0. This is because the line of best fit should pass through the mean of each group. The point that is least furthest from all other points in each group is the mean.

Regression can also be used for hypothesis testing. We can see that there are t-values and p-values in the regression output. If the p-value for intercept is significant, then that tells us that the y intercept is significantly different from 0. If the slope is significant, that tells us the slope is significantly different from 0, and in this case it can be interpreted as the means of the two groups being different.

Visualizing:

plot.default(x=test_df$UDF_text_07, y=test_df$WordCount)
lines(test_df$UDF_text_07, predict(model2),col='blue')

Another way to look at it with boxplots and zoomed in:

plot(x=test_df$UDF_text_07, y=test_df$WordCount,ylim=c(0,3000))
lines(test_df$UDF_text_07, predict(model2),col='blue')

Lets compare that p-value above to a t-test:

t.test(test_df$WordCount~test_df$UDF_text_07)

Hmm. We only passed one x varible in, but here it looks like we are doing regression with 2 x variables.

Sidenote, if you have multiple x variables, each X gets its own estimate for slope. These are usually called betas instead of slope. For example if you had 3 x variables, your equation would be y = m1x2 + m2x2 +m3x3 + b, and linear regression would estimate all 3 m.

The reason it looks like we are doing regression on 2 x variables is because we are. In order to one hot encode a factor with 3 levels, you need two columns.

Visualizing the model and data:

Lets plot our model and data like we did previously:

plot.default(x=d$UDF_text_07,y=d$WordCount)
lines(d$UDF_text_07, predict(model3),col='blue')

And now lets zoom in to see what our line looks like:

plot.default(x=d$UDF_text_07,y=d$WordCount,ylim = c(0,3000))
lines(d$UDF_text_07, predict(model3),col='blue')

Huh? What kind of line is this? How come its splitting like that? Did we break it?

Nope, this is actually correct. Remember, we have 2 x variables and a y variable so that means our data is 3 dimensional. It looks broken because we are viewing it in 2D. Lets see what it looks like in 3D.

Visualizing linear regression with a factor with 3 levels

First we will manually create the one hot encoding:

# Manually create dummy variables:
d$x1 = ifelse(d$UDF_text_07 == 'Retention',1,0)
d$x2 = ifelse(d$UDF_text_07 == 'Sales',1,0)

Now lets make sure the regression output is exactly the same:

# Double check that this is the same as the model we did before with UDF_text_07
model3 = lm(WordCount ~ x1 + x2, data = d)
